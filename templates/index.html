Ultra-lightweight (<25MB) CPU-only TTS. No GPU required. Install: pip install https://github.com/KittenML/KittenTTS/releases/download/0.8/kittentts-0.8.0-py3-none-any.whl

IndexTTS

Zero-shot voice cloning by Bilibili. Requires an isolated venv under engines/index-tts/. Uses the existing Voice Prompts library for reference audio.

Model weights must be downloaded to engines/index-tts/checkpoints/.
Character limit per synthesis chunk (recommended 300–500).
auto, cuda, cuda:0, cpu
Fallback reference audio when no per-speaker prompt is assigned.
1 = fastest (greedy/sampling). 3 = default quality. Higher = slower GPT stage with marginal quality gain.
10–15 = fast, good quality. 25 = default. 50 = max quality. Affects the s2mel stage (biggest bottleneck).
GPT sampling randomness. 0.5–0.7 = consistent/stable. 0.8 = default. 1.0+ = more expressive/varied delivery.
Limits sampling to the top cumulative probability mass. Lower = more conservative token choices. Default: 0.8.
Limits sampling to the top K tokens at each step. Default: 30. Higher = more variety.
Penalises repeated mel tokens. Raise if you hear stuttering or looping artifacts. Default: 10.0.
Hard cap on output length per chunk (~22 tokens/sec). 1500 ≈ 68s. Raise for very long sentences; lower to speed up short chunks.
How many text tokens the model processes per internal pass before splitting. Smaller = faster but may affect prosody at split points. Default: 120.
